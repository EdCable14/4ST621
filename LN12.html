Hypothesis Testing - Other Considerations

Author

Adam Čabla

Return of the Lady Lasting Tea: Power of the test

Let us return to the Lady Lasting Tea. She recognized all four cups with milk poured first out of the eight that were put in front of her, and we could lean on the belief in her. But what was her chance of doing so? If she could recognize it every time, she couldn’t fail the Fischer´s test, but what if she could, from time to time, be mistaken?

For example, what if her true ability to identify the milk first order was 70%? If you recall, the null hypothesis was that she was randomly guessing, so this chance was set at 50 %. Clearly, even if her ability was partial, the null hypothesis should have been rejected, and we want to know what the probability of doing so was. Formulating this in probability terms, we want to know \(P(H_0 \text{ rejected}|H_1\text{ true})\), which we refer to as the power of the test.

To reject the null hypothesis, either the p-value has to be low or the value of the test criterion (TC) has to be in the rejection region. Using the latter, we can rewrite the power of the test as

\[ P(TC \in W_\alpha |H_1) \]

And specifically for the Lady Lasting Tea example, with true ability set at 0.7 as \[P(X=4|\pi=0.7)\]Now, without random guessing, we will not construct this probability as hypergeometric, but approximate it using the binomial distribution \(X\approx Bi(n=4,\pi=0.7)\) and power is \(\approx 0.2401\).

(Real probability calculation would be more complicated and require us to set up not only the true positive probability of 0.7, but also the false positive probability, and think about how she would fit her decision into the limited space of selection of 4 cups if she thought there were, for example, 3 or 5)

So, if her true ability to recognize cup pouring milk first is limited, her winning the test is not a sure thing, but rather a random event with a given probability, which is the power of the test.

Obviously, this probability, the power, is dependent on Bristol´s true ability to recognize the order of the pouring - the better she can, the higher the power is, and the more likely it is for her to pass the test. For example, to pass the test with a 0.8 probability, her true ability expressed via a binomial distribution would have to be around 95%.

Power of the test in the example of the Z-test

The Z-test is a play model due to its simplicity. We assume data come from a normal distribution with \(\sigma\) known. In this case, the pivot to test the hypothesis about the value of \(\mu_0\) can be constructed as a standardized sample mean.

\[ Z=\frac{\bar{X}-\mu_0}{\sigma}*\sqrt{n}\sim N(0;1) \]

This is the null model (probability distribution assuming \(H_0:\mu=\mu_0\)), and the rejection region for the one-sided alternative hypothesis \(H_1:\mu>\mu_0\) will be

\[ W_\alpha=\{z;z\geq z_{1-\alpha} \} \]

Denoting the point alternative hypothesis that is in line with the ordinal alternative hypothesis above as \(H_1:\mu=\mu_1>\mu_0\), we want to calculate the power of the test as

\[ P(Z\geq z_{1-\alpha}|\mu_1)=P\left(\frac{\bar{X}-\mu_0}{\sigma}\sqrt{n}\geq z_{1-\alpha} |\mu_1 \right)=P\left( \bar{X} \geq \mu_0 + \frac{z_{1-\alpha}\sigma}{\sqrt{n}} | \mu_1\right) \]

And with the assumption of \(\mu=\mu_1\), we know that

\[ \bar{X}\sim N\left(\mu_1;\frac{\sigma}{\sqrt{n}}\right) \]

and can calculate the given probability, that is, the power of the test.

Hence…

From this example, we can see three main factors influencing the power of the test, which are generally always present:

1./ Effect size: the larger the difference between the true value and the null hypothesis is, the more powerful the test is.

2./ Sample size: the more observations we have, the more powerful the test is.

3./ Significance level: even though little hidden, the larger the significance level is, the larger the power is as well.

The last one expresses a trade-off between the two possible errors that can be made; the more we expose ourselves to the type I error (incorrectly rejecting \(H_0\) with probability \(\alpha\)), the less we expose ourselves to the type II error (incorrectly not rejecting \(H_0\) with probability \(\beta\), complement of the power) and vice versa.

Practical considerations

Collecting data is never free - it costs money, time, or attention. Additionally, each subsequent observation provides less information than the previous one. This means that, as researchers, we face trade-offs when considering the question: how much data is enough?

Choosing sample size

Choosing the sample size is a crucial yet often overlooked consideration. In practice, we usually resort to the data already collected; that is, we cannot influence this. In the event that we design the research plan, however, we have several alternatives that depend on the question being asked. We can base our approach on the precision of the point estimate (or the width of the desired confidence interval) or decide on the acceptable error rates, particularly the power. However, the costs are always present, so a cost-benefit analysis is the optimal approach (e.g., how much can we increase the power with additional costs).

Target effect size

We can decide on the sample size and the trade-off between \(\alpha\) and \(\beta\), but we cannot decide on the effect size - it is there, it is objective, and we cannot negotiate it.

The true power is never known, because the true effect size is never known!

However, we can circumvent this issue by specifying our effect size of interest: what effect size do we want to be able to detect with a given power? This is a subjective choice, hopefully based on domain expertise. We can also consider different effect sizes of interest and aim for specific power levels for each of them.

Beware, though, that these power calculations are often more of a rough estimate than something precise. For example, we do not know the exact variability, which is necessary for probabilistic calculations, and must be estimated before data collection begins.

With all the issues in mind, do not let the perfect be the enemy of the good.

Regarding the post-hoc analysis, we can still perform power-based calculations, even though the questions differ in nature. For instance, assuming various effects, what were the power levels for these effects based on our data?

We are not planning for the sample size to achieve a given power level; instead, we are using a known sample size (and variability or other information) to analyze what was actually possible to achieve.

Statistical significance

Numerous misconceptions plague statistical hypothesis testing; let’s address some of them.

One of the most common is mixing statistical significance with the practical meaning. As stated earlier, the term “statistically significant” indicates that the null hypothesis has been rejected. It does not inform us whether this is practically meaningful or not.

We can reject the null hypothesis even if the true effect is negligible for us, simply due to the virtue of a large sample size, or due to a nonsensical specification of the null hypothesis to begin with.

Conversely, we might not achieve a statistically significant result, even if the effect size is large, simply because we lack sufficient power due to a small sample size. In fact, small studies with small sample sizes are often a waste of resources simply because they cannot find the effect, even if the effect would be deemed practically relevant.

A statistically non-significant result is not proof that the null hypothesis is true. That would be assuming that evidence of absence is equivalent to the lack of evidence.

On the other hand, following Fischer´s original idea of significance testing, if we conduct several well-designed experiments with sufficient power to reject the null hypothesis for a practically meaningful effect size, this has a corroborative meaning. It does not prove the null hypothesis, as it cannot be done, but it allows us to place it in the area reserved for well-tested and not rejected hypotheses. It was put into mortal danger and survived from a Popperian perspective.

As a general rule, statistical significance is necessary for preventing spurious decisions, but it is not sufficient from a practical point of view.

Understanding p-value

The p-value is the probability of observed or more extreme data assuming the null hypothesis is true. The counterfactual nature of statistical hypothesis testing can often be confusing. If the null hypothesis is true, p-values are just randomly fluctuating numbers between 0 and 1 (they are uniformly distributed). The p-value is not the probability of the null hypothesis being true (it is or it is not, the probability is thus 1 or 0); that would require a Bayesian approach.

Additionally, the term borderline significance has begun appearing in published literature suspiciously. It simply means the null hypothesis was not rejected, but it could have been with a different (higher) specification of the \(\alpha\) level. But this is a slippery slope, because from the borderline perspective, p-values 0.047 and 0.053 are the same, yet the term is somehow reserved only for the latter.

Attacks on p-values and statistical testing in general

The last two decades have seen a significant increase in opposition to p-values, even though most of them appear to be misinterpretations, in my opinion.

One thing to note is that p-values do not depend solely on the formally expressed null hypotheses, but also on the general “hidden” assumptions, such as the central limit theorem being satisfactory. This is correct, but there is no quantitative inferential procedure that can circumvent this. All of them require some assumptions and are, to some degree, dependent on their fulfillment.

The largest set of criticism, however, is about something different: dishonesty in the research. Let´s first briefly define two of the most common dishonest techniques (not direct fraud) in hypothesis testing:

P-hacking refers to the practice of conducting multiple analyses in an attempt to find statistically significant results. P-hackers use the fact that there is a nonzero error rate to find spurious findings and build their publications (and fame) around these.

HARKing, an acronym for Hypothesis After Research Known, is a technique that involves conducting exploratory analysis first and then pretending it was actually confirmatory analysis - a hypothesis test.

Why do researchers do that? Because they are pressed to publish, and because statistically significant results have a higher chance of being published. The reasoning is that if we refrain from testing, we can address the bias known as publication bias.

But, this is my personal opinion, we cannot. If we change the game, fraudsters and impostors will find new ways to play it to their advantage. As long as publications and citations are cherished, they will be targeted.

Closing remarks

Statistical hypothesis testing is a process that is closely related to the broader concept of hypothesis testing. It is best nested in the critical rationalist approach to science, and there is no need for it to be used for anything other than, well, hypothesis testing. If done correctly, it provides modest yet honest service to those in the business of generating knowledge or making informed decisions.

This, however, requires following correct and quite strict procedural rules, which are here for our protection. If you want to do it safely, follow them.

Hypotheses should be prespecified, optimally pre-registered. In the public domain, analysis plans, data, and codes should be shared. Deviations should be honestly discussed. Low-power studies should generally be avoided or at least interpreted with caution.

Ultimately, it is all about putting the hypotheses, both the null and the alternative, to the test.
